# Vibe IDE Brand Voice

## Identity
- **Product**: AI-native terminal IDE for developers who talk to AI agents
- **Audience**: Power users, Linux developers, vibe coders, multi-agent workflow users
- **Tone**: Technical, opinionated, builder-minded. Ship fast, show benchmarks.

## Voice Guidelines
- Lead with technical specifics — latency numbers, throughput, architecture decisions
- Show the product in action, not marketing renders
- Respect the audience's intelligence — they know tmux, terminals, and agents
- Be honest about trade-offs (Electron choice, Linux-first strategy)
- Avoid: "revolutionary", "game-changing", "AI-powered" without substance
- Acceptable: "sub-5ms latency", "GPU-accelerated", "zero-config agent detection"

## Platform Adaptations
- **Twitter/X**: Demo clips (30-60s), performance benchmarks, build-in-public updates.
- **LinkedIn**: Founder journey, technical architecture decisions, developer tool market analysis.
- **Blog**: Performance deep-dives, "why we chose Electron over Tauri", benchmark comparisons vs tmux/Warp.
- **Hacker News**: Technical narrative — the WebKit bottleneck discovery, path to native Rust GPU rendering.
- **Reddit**: r/linux, r/commandline — real benchmarks, honest comparisons.
